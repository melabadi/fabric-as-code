# Fabric‑as‑Code

### Deploy Microsoft Fabric end‑to‑end — from nothing to a running analytics platform

Azure CLI · Bicep · Fabric REST APIs

---

## The problem

Microsoft Fabric has **two control planes**, and clicking through the portal
covers neither in a repeatable way:

| Layer | What it manages |
| --- | --- |
| **Azure** | The Fabric **capacity** (`Microsoft.Fabric/capacities`) — billable compute |
| **Fabric** | **Workspaces** and **items** (Lakehouse, Warehouse, Notebook, Pipeline…) |
| **Data** | Objects *inside* an item — e.g. **stored procedures** in a Warehouse |

The portal/ARM can't create workspaces or items — only the **Fabric REST API** can.

---

## What this repo does

One config file → a fully working environment:

1. **Provision** a resource group + Fabric capacity (Bicep)
2. **Create** a Fabric workspace (REST)
3. **Assign** the workspace to the capacity (REST)
4. **Deploy items** — Lakehouse, Warehouse, Notebook, Data Pipeline (REST)
5. **Deploy stored procedures** into the Warehouse (T‑SQL)
6. *(optional)* **Connect to Git** for ongoing source control

Every step is **idempotent** and **parameter‑driven** — re‑runnable and safe.

---

## Repeatable by design

- All tenant/subscription/secret values live in a git‑ignored **`.env`**
- Duplicate `.env` per environment: `.env.dev`, `.env.customerA`, …
- Same scripts, different config → identical environment in **any tenant / RG**
- Service‑principal auth for **CI/CD** and **multi‑tenant** automation
- **PowerShell** and **bash** versions of every script

## What you can build on this

- **Medallion data platform** — Lakehouse (bronze/silver/gold) + Spark notebooks
- **SQL warehousing** — Warehouse with versioned schema, tables & stored procs
- **Orchestration** — Data Pipelines chaining notebooks, copies & procs
- **Promotion** — Git integration + deployment pipelines (dev → test → prod)
- **Governance as code** — capacity sizing, admins, and RBAC in source control

## Get started

```bash
git clone https://github.com/<owner>/fabric-as-code
cd fabric-as-code
cp .env.example .env        # fill in tenant / subscription / capacity
pwsh ./scripts/powershell/deploy-all.ps1   # or ./scripts/bash/deploy-all.sh
```

Tear it all down (capacity + workspace) with `99-teardown`.
